Deleted Email Recovery

By default, any email deleted from your mailbox goes through the following process:

  1. The email is moved to your Deleted Items folder.
  2. If you purge or remove items from your Deleted Items folder, they are moved to the Dumpster where they are still recoverable.
  3. The Dumpster only holds email messages for 14 days once they've been removed from Deleted Items.
  4. Deleted email can be recovered from the Deleted Items folder or the Dumpster. Once it has been permanently deleted from the dumpster, it cannot be recovered.

How to Recover Deleted Items from the Dumpster:

  1. Go to the Deleted Items folder.
  2. Under the Home tab, choose Recover Deleted Items from Server.
  3. Choose the email that needs to be recovered (you can choose more than one).
  4. Choose Restore Selected Items and click OK.

Note: Deleted email can be recovered from the Deleted Items folder or the Dumpster. Once it has been permanently deleted from the dumpster, it cannot be recovered. Also, Deleted Email Recovery is only available in OWA or Outlook.

How to Obtain Knowledge Contributor Access for ABC KB

To request Knowledge Contributor access for the ABC KB, follow these steps:

  1. Access the ABC IT Service Portal through the following link: Service Catalog - ABC IT Service Portal.
  2. Submit a request in SNOW with the following details:
    • Request Type – Single User
    • Tenant – Knowledge Management – ABC
    • Persona – Knowledge Contributor
  3. Complete the required trainings. Once the request is submitted and approved, an email containing links to the required trainings will be sent to you. Alternatively, you can access the Knowledge Management Curriculum at Learning Management System, search for Knowledge Management Handbook, complete the trainings, and attach the certificate in the request that is created. Allow 24 hours for the access to apply.

For further assistance in creating articles, refer to the following links:

  • ABC KM Authoring Guide – provides templates, tools, and detailed instructions on common activities.
  • Video Instructions – How to create a knowledge article in ServiceNow

H Drive Read and Write Access

A. Read Access – This is by default provided to users with H drive provisioning.

B. Write Access – After the OneDrive migration, write access is no longer default for users.

In the default setup, Home Drive has limited access permissions, specifically Read Only (R/O). It is not possible to store any data on Home Drive (H:), except for the folders that have been approved for Application data, which include System and Explorer/Favorites. These approved folders grant Read and Write (R/W) access.

Additional information can be found here: H: Drive Permissions

Examples of application folders are:

  • H:\DATA\SapLogon
  • H:\ESOPS

Request for access to H drive application folders can be placed here: H: Drive Permissions Request Form
